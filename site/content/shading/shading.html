"A Shader is a user-defined program designed to run on some stage of a graphics processor. Shaders provide the code for certain programmable stages of the rendering pipeline. They can also be used in a slightly more limited form for general, on-GPU computation." - Khronos.org

An introduction to shading

Shaders are programms written in GLSL (GL Shading Language) which is a dialect of the C programming language. They are used to control the rendering process of 3D sceneries and are applied in multiple steps. These are the shaders available for application during the rendering process:

  • Vertex Shader (required)
  • Tessellation Shader
  • Geometry Shader
  • Fragment Shader (required)
  • Compute Shader

The shaders listed above are ordered in the way they are to be applied during the rendering process. The following page however will only elaborate on the functionality of the Vertex and Fragment Shader, since the others are optional.

Other steps in rendering

Besides shaders there are also a few other important steps in the rendering pipeline which will be explained below:

Vertex and Index Lists

A Vertex List is a list of all vertices in the scene and contains each vertex only once to prevent bloat. A Vertex List is generally very long, given that a simple object like a cube already contains eight vertices. Generally, the more complex the Mesh the more vertices are there to be transmitted to the GPU, the longer the list becomes.

An Index List always has a length of a multiple of three since it contains the data on which vertices are to be connected to form a triangle. An Index List is helpful in minimizing required memory and transfer times because an Index is an Integer while a vertex consists of three coordinates and information on the vertexes color, and it prevents duplicates in the Vertex List.

Rasterization

Rasterization is the conversion of a 3D scene into a 2D pixelated image. After the rasterization of the scene the following shaders are being applied pixel by pixel instead of per object in the scene.

Frame Buffer

The frame buffer is the latest rendered image stored on the GPU. Whenever the screen refreshes it pulls the currently buffered frame. This can also cause an effect known as screen tearing in which two different frames are being displayed simultaneously, causing a tear in the middle of the display. To prevent this a technology called VSync is utilized. This adds a second frame buffer. One frame buffer contains a rendered image while the other is written to. Once the monitor pulls an image from the first buffer the two buffers are swapped and another rendered image is ready immediately. This prevents the GPU from updating the frame buffer while the monitor is pulling, which prevents screen tearing but in turn also increasing latency.

The Vertex Shader is provided with Vertex Attribute data and calculates every vertex in a given field of view. They can also be used to process per-vertex lighting or to perform setup work for later stages of shader application.

The Fragment Shader is a type of shader used to calculate the color of a single pixel or Fragment after it has been generated by the rasterization process.
